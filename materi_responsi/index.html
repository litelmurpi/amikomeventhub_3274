Bab 01

MVC & Arsitektur Laravel

Menjelaskan alur kerja Model-View-Controller (MVC) pada Laravel 12 dan pemetaan struktur foldernya.

1.1 Pola MVC (Model-View-Controller)

Laravel merupakan framework PHP yang menganut arsitektur MVC secara penuh. Pola ini memisahkan logika program menjadi 3 bagian utama:

  • Model: Berinteraksi langsung dengan database. Diwakili oleh Eloquent ORM. Class model diletakkan di direktori app/Models/.
  • View: Mengelola representasi grafis / HTML yang dikirim ke browser pengguna. Dibuat menggunakan Blade Template Engine di dalam direktori resources/views/.
  • Controller: Bertindak sebagai pengatur logika utama aplikasi. Menerima data masukan dari user, meminta data dari Model, memproses logika bisnis, dan mengirimkannya ke View. Terletak di app/Http/Controllers/.

1.2 Struktur Folder Terpenting

Struktur folder penting Laravel 12 yang wajib dipahami di dalam ujian responsi:

  ├── Http/Controllers/ // Pengatur alur request, validasi data, upload, dll.

Implementasi Fitur Search

Menjelaskan fitur pencarian data menggunakan query LIKE/ILIKE dan kompatibilitas driver database.

9.1 Search dengan LIKE & Kompatibilitas Database

Aplikasi EventHub mendukung dua driver database sekaligus: MySQL dan PostgreSQL. Karena PostgreSQL bersifat case-sensitive untuk operator LIKE, maka kita perlu menggunakan ILIKE khusus untuk PostgreSQL:

app/Http/Controllers/Admin/CategoryController.php
public function index(Request $request) {
    $query = $request->input('search');
    $operator = DB::connection()->getDriverName() === 'pgsql' ? 'ilike' : 'like';

    $categories = Category::when($query, function($q) use ($query, $operator) {
        return $q->where('name', $operator, '%' . $query . '%');
    })->latest()->get();
}

9.2 Optimasi Input Pencarian dengan Debounce (JavaScript di Blade)

Ketika membuat fitur pencarian secara real-time (pencarian berjalan otomatis ketika user mengetik), kita harus menghindari pengiriman request ke server pada setiap ketukan tombol (keystroke). Hal ini karena jika user mengetik 10 karakter dengan cepat, browser akan mengirimkan 10 request berturut-turut, yang dapat membebani database.

Untuk mengatasi masalah ini, kita menerapkan teknik Debounce. Debounce menunda eksekusi submit form pencarian hingga user selesai mengetik selama kurun waktu tertentu (misalnya 500 milidetik). Berikut adalah implementasi riil yang digunakan pada halaman admin events:

resources/views/admin/events.blade.php
let searchTimeout = null;
const searchInput = document.getElementById('search-input');
const searchForm = document.getElementById('search-form');

if (searchInput && searchForm) {
    searchInput.addEventListener('input', function() {
        clearTimeout(searchTimeout);
        searchTimeout = setTimeout(() => {
            searchForm.submit();
        }, 500); // Debounce 500ms
    });

    // Menjaga kursor tetap berada di akhir input saat reload
    if (searchInput.value) {
        const val = searchInput.value;
        searchInput.value = '';
        searchInput.focus();
        searchInput.value = val;
    }
}

Penjelasan Alur Logika:

  • Setiap kali ada input baru (ketukan keyboard), clearTimeout(searchTimeout) dipanggil untuk membatalkan rencana pengiriman form sebelumnya.
  • Sebuah timer baru diset menggunakan setTimeout() selama 500 milidetik. Jika user berhenti mengetik selama 500ms, fungsi callback berjalan dan memicu searchForm.submit() untuk memicu submit form pencarian ke database.
  • Trik searchInput.value = '' lalu memfokuskannya dan mengembalikan nilainya kembali digunakan agar kursor teks browser tetap dipaksa berada di karakter paling akhir setelah halaman direfresh/dirender ulang pasca pencarian.

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration {
    public function up(): void {

4 Membuat PartnerController (Lengkap)

Controller ini menangani seluruh operasi CRUD partner, termasuk dual-mode upload (file lokal + URL eksternal) dan cleanup file saat update/delete:

app/Http/Controllers/Admin/PartnerController.php
namespace App\Http\Controllers\Admin;

use App\Http\Controllers\Controller;
use App\Models\Partner;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\DB;
use Illuminate\Support\Facades\File;

class PartnerController extends Controller {

    // Tampilkan daftar partner + fitur search
    public function index(Request $request) {
        $query = $request->input('search');
        $operator = DB::connection()->getDriverName() === 'pgsql' ? 'ilike' : 'like';
        $partners = Partner::when($query, function($q) use ($query, $operator) {
            return $q->where('name', $operator, '%' . $query . '%');
        })->latest()->get();
        return view('admin.partners.index', compact('partners'));
    }

    // Form tambah partner baru
    public function create() {
        return view('admin.partners.create');
    }

    // Proses simpan partner baru (dual-mode: file ATAU url)
    public function store(Request $request) {
        $request->validate([
            'name'     => 'required|string|max:255',
            'logo'     => 'nullable|file|mimes:jpeg,png,jpg,gif,svg|max:2048',
            'logo_url' => 'nullable|url|max:2048',
        ]);

        // Minimal salah satu harus diisi
        if (!$request->hasFile('logo') && !$request->filled('logo_url')) {
            return back()->withErrors(['logo' => 'Harap unggah logo atau masukkan URL.']);
        }

        $logoUrl = '';
        if ($request->hasFile('logo')) {
            $file = $request->file('logo');
            $fileName = time() . '_' . uniqid() . '.' . $file->getClientOriginalExtension();
            $file->move(public_path('uploads/partners'), $fileName);
            $logoUrl = 'uploads/partners/' . $fileName;
        } else {
            $logoUrl = $request->logo_url;
        }

        Partner::create(['name' => $request->name, 'logo_url' => $logoUrl]);
        return redirect()->route('admin.partners')->with('success', 'Partner berhasil ditambahkan.');
    }

    // Hapus partner + cleanup file lokal
    public function destroy(Partner $partner) {
        if ($partner->logo_url && !str_starts_with($partner->logo_url, 'http')
            && File::exists(public_path($partner->logo_url))) {
            File::delete(public_path($partner->logo_url));
        }
        $partner->delete();
        return redirect()->route('admin.partners')->with('success', 'Partner dihapus.');
    }
}

Bab 14

Rendering Partner di Halaman Publik

Menjelaskan bagaimana data partner ditampilkan di halaman depan (welcome page) dengan efek visual dinamis.

1 Mengambil Data Partner di HomeController

Data partner diambil di HomeController lalu dikirim ke view welcome:

app/Http/Controllers/HomeController.php
use App\Models\Partner;

public function index() {
    $events = Event::with('category')->get();
    $partners = Partner::latest()->get();   # Ambil semua partner, urutkan terbaru

    return view('welcome', compact('events', 'partners'));
}

Fungsi compact('events', 'partners') akan membuat array asosiatif: ['events' => $events, 'partners' => $partners] yang tersedia sebagai variabel di Blade.

2 Rendering Logo Partner dengan Conditional Source

Karena logo bisa berasal dari URL internet atau file lokal, kita perlu mengecek tipe sumbernya:

resources/views/welcome.blade.php
<!-- Partners Section -->
<section class="max-w-7xl mx-auto px-6 py-20">
  <h2>Partner & Sponsor Kami</h2>

  <div class="grid grid-cols-2 md:grid-cols-4 gap-8">
    @foreach($partners as $partner)
    <div class="group flex flex-col items-center p-6">
      <img
        src="{{ Str::startsWith($partner->logo_url, 'http')
               ? $partner->logo_url           <!-- URL internet → gunakan langsung -->
               : asset($partner->logo_url) }}" <!-- File lokal → bungkus dengan asset() -->
        alt="{{ $partner->name }}"
        class="grayscale group-hover:grayscale-0 transition">
      <span>{{ $partner->name }}</span>
    </div>
    @endforeach
  </div>
</section>
3 Teknik Grayscale-to-Color Hover

Class Tailwind CSS grayscale membuat logo tampil abu-abu secara default. Saat user menghover card parent (group), class group-hover:grayscale-0 mengembalikan warna asli logo secara dinamis:

Penjelasan CSS Utility
/* grayscale       → CSS filter: grayscale(100%) */
/* grayscale-0     → CSS filter: grayscale(0%)   */
/* group           → Parent container untuk group-hover */
/* group-hover:*   → Aktif saat parent ".group" di-hover */
/* transition      → Animasi halus antar state */

Teknik ini menciptakan efek visual elegan: logo partner tampil hitam-putih saat idle, dan berwarna saat disorot — membuat halaman terlihat lebih bersih dan profesional.
